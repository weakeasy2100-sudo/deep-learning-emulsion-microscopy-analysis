## 1. Current Project Purpose

A staged emulsion-like microscopy image analysis workflow under limited annotation conditions,
built with synthetic and AI-generated microscopy-like images.

Stages completed:
- **V1** — clean synthetic baseline (classical detection + SimpleCNN)
- **V2** — realistic synthetic data (noise, uneven background, edge-cutting, dust)
- **V2.1** — high-density small-droplet data (crowded, small, low-contrast)
- **V3** — generalization stress test (full pipeline, not CNN alone)
- **V4** — mixed-data training (combined V1+V2+V2.1 with augmentation)
- **V5** — patch quality diagnosis (detector/cropping bottleneck analysis)
- **V6** — detector experiment (BG-sub + CLAHE + local threshold, conservative)
- **V7** — confidence-aware uncertainty analysis (softmax confidence, not validated accuracy)

The project demonstrates workflow capability, robustness testing, and failure-mode diagnosis.
It does NOT claim validated accuracy on real microscopy images.
All images are synthetic — generated by code with known ground-truth labels.

## 3. V1 — Core Pipeline (read-only)

### Step 1 — Synthetic Image Generation (`src/generate_data.py`)

**Input:** None (self-contained, NumPy random seeds only)

**Parameters:**
- 150 images: 50 per class (small / medium / large)
- Image size: 256×256 px, grayscale
- Droplets per image: 4–9
- Radius ranges: small 6–14 px, medium 15–25 px, large 26–40 px
- Seed: `idx * 137 + 42`

**Outputs:**
| Path | Description |
|------|-------------|
| `data/raw/<class>_<NNN>.png` | 150 synthetic grayscale PNG images |
| `data/metadata.csv` | image_filename, droplet_count, mean_radius, size_class, random_seed |
| `results/synthetic_examples.png` | 3×5 preview grid (one row per class) |

---

### Step 2 — Classical Image Analysis (`src/classical.py`)

**Pipeline (`detect_droplets()`):**
1. Gaussian blur (σ = 1.5)
2. Otsu threshold → binary mask
3. Remove small objects + fill small holes (min area = 30 px)
4. `clear_border` — discard droplets touching image edge
5. `regionprops` — diameter 8–100 px, circularity ≥ 0.50

**Outputs:**
| Path | Description |
|------|-------------|
| `results/classical_detection_overlay.png` | 3×3 detection overlay grid |
| `results/droplet_size_distribution.png` | Histogram + violin plot per class |

**Public API (imported by all downstream scripts):**
- `detect_droplets(img)` — used by `src/dataset.py`, and in-memory by V3, V5, V6, V7
- `CLASS_ORDER`, `COLORS` — used by `src/dataset.py`, `src/train_classifier.py`

---

### Step 3 — SimpleCNN Classification

**Scripts:** `src/dataset.py`, `src/model.py`, `src/train_classifier.py`

**SimpleCNN architecture:**
```
Conv(1→16, 3×3, pad=1) → ReLU → MaxPool(2)   # → 16×32×32
Conv(16→32, 3×3, pad=1) → ReLU → MaxPool(2)  # → 32×16×16
Flatten → Linear(8192→128) → ReLU → Dropout(0.3) → Linear(128→3)
```
- Adam (lr = 1e-3), CrossEntropyLoss, 20 epochs, batch size 32
- Stratified split: 70/15/15, seed = 42
- Patch size: 64×64 px, reflect-padded if near image edge
- Normalisation: `ToTensor` + `Normalize(mean=0.5, std=0.5)`

**Outputs:**
| Path | Description |
|------|-------------|
| `data/patches/<class>/` | 64×64 PNG patches (not committed) |
| `results/simple_cnn.pth` | Trained model weights (state dict) |
| `results/training_curve.png` | Train/val loss + val accuracy over 20 epochs |
| `results/confusion_matrix.png` | Confusion matrix on held-out test set |
| `results/prediction_examples.png` | 3×5 patch grid with true/predicted labels |

## 6. V3 — Generalization Stress Test (`src/stress_test_generalization.py`)

Tests `results/simple_cnn.pth` (V1 model) on all three datasets.

> **Pipeline note:** This test evaluates the full pipeline (classical detection → patch cropping →
> CNN classification), not the CNN alone. Performance drops may reflect detector failure, poor patch
> cropping, domain shift, or CNN classification limitations — or a combination of all of these.

**Design decisions:**
- V1 patches: optionally read from `data/patches/` using `make_splits(seed=42)` — no re-detection
- V2/V2.1: in-memory patch extraction only — nothing written to `data/`
- `pred_store` dict caches (all_true, all_pred) per dataset — no double inference

## 7. V4 — Mixed-Data Training (`src/train_mixed_classifier.py`)

Trains a new SimpleCNN on combined V1 + V2 + V2.1 data with augmentation.
Saves to `results/simple_cnn_mixed.pth`. Never overwrites `results/simple_cnn.pth`.

**Augmentation transform (training only):**
```python
RandomHorizontalFlip(p=0.5)
RandomRotation(degrees=15)
ColorJitter(brightness=0.3, contrast=0.3)
GaussianBlur(kernel_size=3, sigma=(0.1, 1.5))
ToTensor() + Normalize(mean=[0.5], std=[0.5])
```

**Evaluation transform:** `ToTensor() + Normalize(mean=[0.5], std=[0.5])` (no augmentation)

**Training:** 30 epochs, batch size 32, Adam lr=1e-3, best-val-accuracy checkpoint.

**Dataset assembly:**
- V1: `make_splits(seed=42)` → re-wrap `.pairs` with augment/eval transform
- V2/V2.1: in-memory `_extract_patches_raw()` → `_split_pairs()` → `_InMemoryPatchDataset`
- Train sets combined via `ConcatDataset`

**Outputs:**
| Path | Description |
|------|-------------|
| `results/simple_cnn_mixed.pth` | Mixed-trained model weights |
| `results/mixed_training_curve.png` | Train/val loss + accuracy over 30 epochs |

## 8. V5 — Patch Quality Diagnosis (`src/analyze_patch_quality.py`)

Diagnoses whether the performance bottleneck is the CNN or upstream detection and cropping.

**Per-patch metrics:**

| Metric | Definition | Interpretation |
|--------|------------|----------------|
| `mean_brightness` | Mean pixel intensity (0–255) | Dark patches may indicate failed crops |
| `std_dev` | Pixel standard deviation | Low std → featureless patch, likely a non-droplet region |
| `center_surround_ratio` (CSR) | Mean of centre 16×16 / mean of full 64×64 | CSR ≈ 1.0 → crop not centred on a droplet |

**Diagnosis thresholds used in terminal output:**
- `std_dev < 15` → low-quality patch warning
- `CSR > 1.05` → likely misaligned crop warning

**V1 asymmetry:** reads `data/patches/` directly (no re-detection). V2/V2.1 use in-memory extraction.

**Outputs:**
| Path | Description |
|------|-------------|
| `results/patch_quality_gallery.png` | 3×10 patch gallery per dataset with σ annotation |
| `results/patch_quality_statistics.png` | 2×2 violin plots + detection rate bar chart |

## 9. V6 — Detector Experiment (`src/improve_detector.py`)

Side-by-side comparison of original Otsu detector vs. improved detector.
`src/classical.py` is not modified. This script is experimental only.

**Improved detector pipeline (`detect_droplets_improved()`):**
1. Background subtraction: Gaussian (σ=50), subtract from image, re-centre to 128
2. CLAHE: `equalize_adapthist(clip_limit=0.03)` — conservative
3. Gaussian blur (σ=1.5) — same as original
4. Local adaptive threshold: `threshold_local(block_size=51)` — replaces Otsu
5. Morphological cleanup, `clear_border`, `regionprops` — same as original
6. `IMPROVED_MIN_DIAM = 5` (vs. original 8); circularity ≥ 0.50, max diameter 100 px

**Comparison design:** both detectors run on each image; count and quality metrics collected together.

**Outputs:**
| Path | Description |
|------|-------------|
| `results/detector_comparison_overlay.png` | 3×4 overlay grid (green=original, orange=improved) |
| `results/detector_comparison_counts.png` | 2×2 grouped bars: count, std, CSR, high-CSR fraction |
| `results/improved_patch_quality_gallery.png` | Patch gallery for improved-detector detections |

## 10. V7 — Confidence-Aware Uncertainty Analysis (`src/analyze_prediction_confidence.py`)

**Threshold:** `HIGH_CONF_THRESHOLD = 0.80`

**Adaptive loading:** requires `results/simple_cnn.pth`; loads `results/simple_cnn_mixed.pth` if present for comparison.

> ⚠ Caveat (repeated in all figures and terminal output):
> "High confidence = model certainty, NOT validated accuracy.
> A confidently-wrong prediction is possible without ground-truth labels."

**Per-patch record:**
`dataset`, `model`, `pred_class`, `pred_label`, `max_conf`, `probs` (3-element),
`patch_std`, `patch_csr`, `is_high_conf`, `patch` (shared reference, not copied)

**Inference:** batched (BATCH_SIZE=64), softmax over raw logits.

**Outputs:**
| Path | Description |
|------|-------------|
| `results/confidence_distribution.png` | Histogram per (model×dataset), threshold line, high-conf fraction |
| `results/high_vs_low_confidence_examples.png` | 5 highest + 5 lowest confidence patches per dataset |
| `results/uncertain_patch_gallery.png` | Patches with conf < 0.80, annotated with conf% and pred_label |
| `results/quality_confidence_relationship.png` | Scatter: patch_std vs. max_conf, coloured by pred_class |

## 14. Current Limitations

**V1 (core pipeline):**
- Synthetic data only. High test accuracy reflects controlled, idealized generation conditions.
- No data augmentation in V1 training.
- Overlapping droplets may be missed or merged by the classical detector.
- Border droplets discarded by `clear_border` — reduces patch counts near image edges.
- No transfer learning baseline; no comparison with pretrained models.

**V2/V2.1 (harder synthetic data):**
- Still fully synthetic — not real microscopy images.
- V2.1 size ranges are smaller than V1, creating distribution mismatch with the V1-trained model.

**V3 (stress test):**
- Evaluates the full pipeline, not CNN alone. Performance drops cannot be cleanly attributed
  to a single component without ablation over detection, cropping, and classification separately.

**V4 (mixed training):**
- Mixed training partially improves high-density accuracy but does not fully solve robustness.
- V2 realistic accuracy does not improve consistently.

**V5 (patch diagnosis):**
- CSR is a heuristic, not a ground-truth quality measure.
- Low `std_dev` may reflect a featureless background region or a valid small droplet — ambiguous
  without labels.

**V6 (detector experiment):**
- The improved detector is experimental and is NOT integrated into the main pipeline.
- `src/classical.py` is unchanged.
- More detections ≠ better detections; patch quality metrics are required alongside count.

**V7 (confidence analysis):**
- Softmax confidence ≠ validated accuracy. Confident errors are possible.
- No human-labeled ground truth exists to validate which high-confidence predictions are correct.
- The confidence threshold (0.80) is a design choice, not a calibrated precision cutoff.

## 15. Rule for Future Development

> **Only add new features as optional extensions. Do not overwrite or break MVP v1.**

- **Do not modify** `src/generate_data.py`, `src/classical.py`, `src/dataset.py`,
  `src/model.py`, or `src/train_classifier.py` unless fixing a confirmed bug.
- **Do not rename** any file listed in Section 12.
- **Do not change** any output filename listed in Section 13.
- **Do not overwrite** `results/simple_cnn.pth` or any committed result figure.
- **Do not retrain** the V1 model.
- New models, augmentation pipelines, or evaluation scripts must be added as **new files**
  writing to **new result paths**.
- Before any change, verify that `python src/generate_data.py && python src/classical.py &&
  python src/train_classifier.py` still runs end-to-end without errors.

**Planned future directions (not yet implemented):**
- Integrate open-CMD or another annotated droplet/microparticle dataset
- Evaluate with real ground-truth labels
- Move from patch classification toward instance detection or segmentation
- Compare YOLO / Mask R-CNN / U-Net / SAM-based methods
- Real droplet size measurement validation
- Add manually labeled real microscopy images if available
